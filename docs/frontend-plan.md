# 御链先知前端规划

## 1. 产品定位

御链先知是面向智能合约审计人员、区块链安全团队、Web3 项目开发与风控人员的跨模态语义增强智能合约攻击检测系统。前端需要围绕「交易哈希输入、函数调用流解析、静态语义对齐、向量检索、RAGFlow 推理、风险报告」形成完整闭环。

页面呈现以作品书为核心依据，突出以下能力：

- 基于交易哈希触发链上交易与合约信息解析。
- 还原函数调用顺序、上下文切换边界、参数依赖路径。
- 展示调用流净化、递归子树融合、跨合约调用克隆消除、结构相似调用折叠。
- 呈现静态语义资料、动态行为向量、Top-k 相似攻击路径、Prompt 上下文。
- 输出漏洞类型、风险等级、攻击路径、证据片段、处置建议。
- 展示 ETHBench v2.0 指标：准确率 93.6%、召回率 95.8%、F1 0.947、时延 6.8 秒。

## 2. 设计风格

整体风格参考专业安全运营台与链上浏览器，采用冷静、克制、高密度的信息架构。

- 字体：中文衬线字体族，延续第一份作品的正式感。
- 色彩：浅色底、深墨色主文、蓝绿安全色、红橙风险色，纯色块与线框表达状态。
- 版式：单屏优先，左侧输入与任务状态，中部图谱与分析结果，右侧证据、指标、报告摘要。
- 动效：分析过程使用阶段推进、节点亮起、路径流转、图表增量出现、报告逐段生成。
- 组件形态：少量面板化工具区，减少装饰卡片，用表格、图谱、时间线、代码块、证据列表承载信息。

## 3. 页面结构

### 3.1 作品主页

目标：快速说明作品价值、技术路径、指标优势与应用场景。

首屏内容：

- 品牌：御链先知 / RAGFlow。
- 主张：大模型驱动的跨模态语义增强智能合约攻击检测系统。
- 支撑文本：围绕智能合约复杂攻击链、跨合约调用、动态行为语义、RAG 推理。
- 关键指标：93.6% 准确率、95.8% 召回率、0.947 F1、6.8s 时延。
- 入口按钮：进入系统、查看技术路线。

主页模块：

- 攻击背景：近年链上安全事件、56 亿美元损失、DeFi / NFT / 跨链桥场景。
- 技术流程：交易哈希 -> 调用流提取 -> 语义资料对齐 -> Top-k 检索 -> Prompt 构建 -> 风险报告。
- 核心创新：多模态攻击检测机制、函数调用路径压缩、双层语义匹配向量知识库。
- 指标对比：Slither、TXSpector、SmartKG、GPT-4、RAGFlow 横向对比。
- 应用场景：审计辅助、交易风险预警、协议上线前检测、链上风控监控。

### 3.2 登录 / 注册

目标：形成完整产品入口。

控件：

- 邮箱、密码、邀请码。
- 登录 / 注册切换。
- 系统概览侧栏：链上交易、函数调用、语义检索、风险报告四项能力。

### 3.3 系统工作台

导航建议：

- 工作台
- 调用链
- 语义检索
- 风险报告
- 指标
- 样本库

工作台首屏布局：

- 左侧：交易输入与分析配置。
- 中部：分析流程与当前输出区。
- 右侧：链信息、合约信息、风险摘要、运行指标。

输入方式：

- 交易哈希输入，限制 0x 开头 66 位十六进制。
- 合约地址输入，限制 0x 开头 42 位十六进制。
- 文件上传入口，限制 `.sol`、`.json`、`.txt`、`.log`。
- 样本库选择入口，提供 Uniswap V3、Compound V3、跨链桥、重入攻击、闪电贷攻击等类型。

分析过程阶段：

- 链上数据采集
- 函数调用流提取
- 调用轨迹压缩
- 静态语义对齐
- Top-k 向量检索
- Prompt 构建
- 风险判定

空态：

- 首次进入展示输入区、流程骨架、支持格式、链路状态。
- 中部保留结构化工作区，等待用户输入交易哈希或上传合约材料后生成结果。

### 3.4 调用链页面

目标：把复杂攻击路径可视化。

内容：

- 合约节点图：EOA、代理合约、目标合约、DEX、预言机、跨链桥。
- 函数节点：transfer、approve、swap、borrow、withdraw、callback、exploit。
- 路径标注：跨合约调用、状态变更、资金流向、异常分支。
- 压缩前后对比：
  - Uniswap V3 样本节点数 472 -> 51。
  - 关键路径长度 17.3 -> 17.1。
  - 处理时延 12.4ms -> 5.2ms。
  - F1 0.783 -> 0.901。
- Compound V3 聚合指标：
  - 调用节点数 142 -> 31。
  - 嵌入相似度 0.19 -> 0.83。
  - 分析耗时 4.7s -> 1.5s。

交互：

- 节点点击显示函数签名、参数、调用深度、风险标签。
- 路径高亮显示关键攻击链。
- 压缩开关切换原始调用流与聚合调用流。

### 3.5 语义检索页面

目标：展示 RAGFlow 的可信推理来源。

内容：

- 查询向量摘要：调用路径、参数依赖、状态切换。
- Top-k 相似攻击路径：样本名称、相似度、漏洞类型、证据来源。
- 静态语义资料：反编译摘要、关键函数职责、安全机制、状态变量。
- Prompt 上下文：交易元数据、交易负载、检索证据、静态语义补全。
- 向量库状态：索引规模、命中率、检索时延、语义迁移率。

建议组件：

- 检索结果列表
- 证据片段折叠区
- Prompt 构建步骤条
- 相似度条形图
- 语义对齐矩阵

### 3.6 风险报告页面

目标：输出可用于审计决策的结构化结论。

报告结构：

- 风险等级：高 / 中 / 低。
- 漏洞类型：重入、闪电贷组合攻击、权限绕过、TOCTOU、状态污染、预言机操纵。
- 置信度：模型置信、检索一致性、证据覆盖率。
- 攻击路径：按调用顺序列出关键节点。
- 证据来源：交易哈希、函数调用片段、静态语义资料、相似案例。
- 处置建议：暂停合约交互、检查授权边界、增加状态校验、限制外部调用、补充审计规则。
- 导出动作：导出 PDF、加入样本库、生成审计附件、标记复核。

视觉表达：

- 顶部风险摘要条。
- 中部攻击路径时间线。
- 右侧证据和处置建议。
- 底部报告字段完整性检查。

内容：

## 4. 前端行为规则

- 用户输入或上传后进入逐步分析状态，每个阶段有进度、日志、状态颜色。
- 每次完成分析时生成不同的风险类型、置信度、攻击路径、Top-k 结果和报告摘要。
- 合法交易哈希触发分析；非法输入显示格式错误。
- 合法文件类型触发分析；其他文件类型显示格式限制。
- 工作台、调用链、语义检索、风险报告、样本库共享同一份当前分析结果。
- 首次进入各分析页面保持空态结构，输入完成后填充图表和报告。
- 所有页面避免重复解释型文字，优先用指标、图表、路径、表格、证据列表表达。

## 5. 数据与案例池

建议内置结果池：

- 重入攻击：withdraw -> external call -> callback -> repeated withdraw。
- 闪电贷组合攻击：borrow -> swap -> price manipulation -> repay。
- 权限绕过：delegatecall -> role check bypass -> privileged action。
- TOCTOU：check state -> external interaction -> state changed -> unsafe execution。
- 状态污染：oracle update -> stale price -> collateral extraction。
- 跨链桥攻击：verify proof -> message replay -> asset mint。

每个结果包含：

- 链与协议：Ethereum / BSC / Polygon，Uniswap / Compound / Aave / Bridge。
- 风险等级与置信度。
- 调用节点数组。
- Top-k 相似案例。
- 证据片段。
- 处置建议。
- 指标变化。
