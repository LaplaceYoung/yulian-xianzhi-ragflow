# Project Brief

## Current Problem

- 目标用户：智能合约审计人员、区块链安全团队、Web3 项目开发与风控人员。
- 使用场景：智能合约攻击检测、组合漏洞分析、函数调用轨迹解释、审计辅助决策。
- 核心痛点：智能合约攻击链条跨合约、跨函数、跨语义资料，人工审计和传统规则检测难以覆盖复杂组合漏洞。
- 现有方案缺口：单一静态分析或日志分析难以同时利用调用轨迹、语义资料和大模型推理能力。

## Plan

1. 核心系统：跨模态语义增强智能合约攻击检测系统。
2. 核心 AI 技术：函数调用流压缩、静态语义资料对齐、向量嵌入检索、RAG 与 Prompt 构建。
3. 工程闭环：日志采集、调用轨迹解析、语义资料构建、漏洞检索、检测结果输出。
4. 前端展示方式：样本合约输入、攻击链检测输出、漏洞分布与检测效果展示。

## Success Criteria

- 指标 1：漏洞检测准确率、召回率、误报率。
- 指标 2：调用轨迹压缩效果和检索命中质量。
- 指标 3：检测耗时、样本覆盖度和系统可用性。
- 落地证据：项目研究报告、测试数据说明、检测界面截图、作品视频或可运行入口。

## Risks

- 技术风险：大模型输出稳定性和可解释性需要通过提示词、检索证据和测试集约束。
- 数据风险：漏洞样本来源、分类标签和攻击链标注质量会影响评审可信度。
- 查新风险：需要把 RAG、调用流压缩、跨模态语义增强拆成可检索创新点。
- 前端展示风险：检测链路较长，需要准备稳定样例和离线展示材料。
